# Joblynk.live Deployment Master Guide ## 1. Local Development - Install XAMPP, start Apache & MySQL. - Create folder `C:\xampp\htdocs\joblynk` and copy all project files. - Create database `joblynk` in phpMyAdmin, import SQL schema. - Edit `includes/db.php` with local credentials (root, empty password). - Test at `http://localhost/joblynk`. ## 2. GitHub Setup - `cd C:\xampp\htdocs\joblynk` - `git init` - Create `.gitignore` with: includes/db.php, uploads/, .vscode/, *.log - `git add .` and `git commit -m "Initial commit"` - Create repo on GitHub (no README), then: ## 3. Choose Hosting (PHP + MySQL) - Sign up at Hostinger, Bluehost, A2 Hosting, etc. - Get FTP/cPanel access. ## 4. Deploy Files - **Option A (FTP):** Upload all files except `includes/db.php` to `public_html`. - **Option B (Git):** Use cPanel Git Version Control to clone repo into `public_html`. ## 5. Remote Database - Create database in cPanel, note name/user/password. - Import local SQL into remote database. - Create `includes/db.php` with remote credentials and upload manually. ## 6. Permissions - Folders: 755, Files: 644. - Ensure `uploads/` is writable (755 or 775). ## 7. Cron Job (Job Alerts) - In cPanel Cron Jobs, add daily command: `/usr/bin/php /home/username/public_html/cron/send-alerts.php >> /home/username/logs/alerts.log 2>&1` ## 8. Domain - Point domain to hosting (update nameservers or A record). - Enable SSL (free via Let's Encrypt). ## 9. Test Everything - Register, login, post jobs, apply, save, review, payments. ## 10. Maintenance - Never commit `db.php`. - Backup regularly. - Pull updates: `git pull origin main` on server (if using Git deployment).
